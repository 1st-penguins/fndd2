# 로컬 테스트 가이드 (로그인 없이 확인하기)

배포 전 로컬에서 문제풀이·과목 페이지를 **로그인 없이** 확인하는 방법입니다.

---

## 1. URL에 `?test=1` 붙이기 (가장 간단)

로컬(`localhost`, `127.0.0.1`)에서만 동작합니다.

- **메인:**  
  `http://127.0.0.1:3000/index.html?test=1`
- **문제풀이(과목별, 20문항):**  
  `http://127.0.0.1:3000/exam/2025_운동생리학.html?test=1`
- **모의고사(80문항):**  
  `http://127.0.0.1:3000/exam/2025_모의고사_1교시.html?test=1`
- **과목 선택 페이지:**  
  `http://127.0.0.1:3000/subjects/subject_운동생리학.html?test=1`

`?dev=1` 도 같은 의미로 사용할 수 있습니다.  
프로덕션(실서버)에서는 이 파라미터는 **무시**되며, 로컬이 아니면 개발 모드로 인정하지 않습니다.

---

## 2. 개발 모드 키 사용 (DEV 버튼)

1. 로컬에서 **메인 페이지** 접속: `http://127.0.0.1:3000/index.html`
2. 우측 상단 **DEV** 버튼 클릭
3. 키 입력: **4578** 입력 후 확인
4. 페이지가 새로고침되면 개발 모드 ON → 로그인 없이 모든 메뉴·문제풀이 이용 가능
5. 이후 **같은 브라우저**에서 exam, subjects 등 다른 페이지로 이동해도 로그인된 것처럼 동작

키는 `localStorage`에 저장되므로, 탭/창을 닫지 않는 한 유지됩니다.  
다른 관리자에게는 “로컬에서만 DEV 버튼 + 4578”만 알려주면 됩니다.

---

## 3. 로컬 서버 띄우기

```bash
# Python
cd 프로젝트폴더
python -m http.server 3000
```

브라우저에서 `http://127.0.0.1:3000/` 로 접속한 뒤, 위 1번 또는 2번 방법으로 테스트하면 됩니다.

---

## 요약

| 방법              | 용도                         |
|-------------------|------------------------------|
| **?test=1** 붙이기 | 로컬에서 바로 한 페이지만 테스트 |
| **DEV + 4578**    | 로컬에서 메인부터 전체 플로우 테스트 |

둘 다 **로컬에서만** 적용되며, 배포 URL에서는 적용되지 않습니다.
