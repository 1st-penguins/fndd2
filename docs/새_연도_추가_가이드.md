# 새 연도 추가 가이드 (건강운동관리사)

새 연도의 건강운동관리사 기출문제를 퍼스트펭귄에 추가할 때 수정해야 하는 모든 파일과 순서를 정리한 문서입니다. 2018년도 도입 작업(2026-04-11)을 기준으로 작성되었습니다.

> 스포츠지도사 1급·2급(`si1.html`, `si2.html`, `exam-new-sports*/`, `data/sports*/`) 경로는 이 가이드의 대상이 아닙니다. 자격증별로 디렉터리 구조가 분리되어 있으므로 혼동 주의.

---

## 선행 조건 (작업 시작 전 확보할 자료)

1. **정답지 원본** — `data/건강운동관리사_최종정답_2018-2025.docx` 같은 공식 정답 표. "전체", "1,3" 같은 복수/전체 정답 표기 포함.
2. **문제 이미지 파일 20장 × 8과목 = 160장** — 파일명은 반드시 `문제 (1).png` ~ `문제 (20).png` (공백·소문자 확장자 고정).
3. **이미지 폴더** — `images/{YYYY} {과목명}/` 정확히 이 형식. 과목명 8개는 다음과 같이 고정(JSON 경로와 1:1 매칭):
   - `건강체력평가`, `기능해부학`, `병태생리학`, `스포츠심리학`, `운동부하검사`, `운동상해`, `운동생리학`, `운동처방론`
   - ⚠️ 정답지 docx 등에는 `건강·체력평가`로 표기되지만 실제 폴더/파일명은 점(·) 없이 `건강체력평가`로 통일.
4. (가능하면) **각 문제의 해설 원본** — 없으면 이미지를 Claude 비전으로 읽어 직접 작성.

## Phase 1 — 데이터 작성 (JSON 8개)

### 1-1. JSON 파일 위치
`data/{YYYY}_{과목명}.json` 형식. 총 8개 파일.

### 1-2. 스키마 (필수 필드)

각 문항은 다음 6개 필드를 **모두** 포함해야 합니다.

```json
{
  "id": 1,                              // 1~20 순차
  "questionImage": "images/2018 운동생리학/문제 (1).png",
  "correctAnswer": 0,                   // 0-based (1번 → 0)
  "tags": ["지방분해","호르몬","..."],    // 5~8개 권장
  "questionText": "문제 본문...\n① 선택지1\n② 선택지2\n③ 선택지3\n④ 선택지4",
  "explanation": "이 문제의 정답은 1번입니다. <br/> sol) <br/> ..."
}
```

### 1-3. 정답 인코딩 규칙

| 원본 표기 | JSON `correctAnswer` |
|---|---|
| `1` | `0` |
| `2` | `1` |
| `3` | `2` |
| `4` | `3` |
| `1,3` (복수) | `[0, 2]` |
| `2,4` | `[1, 3]` |
| `1,2,4` | `[0, 1, 3]` |
| `전체` (모두 정답) | `[0, 1, 2, 3]` |

### 1-4. 해설 스타일 (기존 연도와 통일)

기존 2019~2025 해설의 톤/포맷을 반드시 따라야 합니다. 가장 최신인 2025 JSON이 기준 참고용.

**기본 템플릿**
```
이 문제의 정답은 N번입니다. <br/> sol) <br/>
✅ [핵심 개념 제목] <br/>
 - [설명] <br/>
 - [설명] <br/>
💡 [요약·팁] <br/><br/>
 📘 오답 분석 <br/>
① [왜 틀렸는지]. <br/>
② [왜 틀렸는지]. <br/>
```

**사용 기호**
- `✅` sol) 내 핵심 개념 헤더
- `📌` 보조 설명·하위 카테고리
- `💡` 요약·핵심 팁
- `👉🏻` 결론·정답 방향
- `⚠️` 경고·전원정답
- `📘 오답 분석` 섹션 구분자
- `<br/>` 줄바꿈 (HTML `<br>` 아님)

**복수/전체 정답 표준 문구**
- 복수: `이 문제의 정답은 1,3번입니다.` 로 시작
- 전체: `이 문제의 정답은 1번, 2번, 3번, 4번입니다. <br/> sol) <br/> ⚠️ 전원정답 처리된 문제 — [사유]`

**오답 분석 규칙**
- "옳지 않은 것 고르기" 문제: 정답(틀린 선지) **제외**, 나머지 선지에 "① ... 올바른 설명이다." 형식
- "옳은 것 고르기" 문제: 틀린 선지에 대해 "② ... 잘못된 설명이다." 형식
- 보기형(ㄱㄴㄷㄹ): `ㄱ. (O)/(X) [이유]`
- 참/거짓형: 각 보기 ○×와 이유

### 1-5. questionText 작성 규칙
- 번호 접두사는 과목별로 기존 연도와 **동일한 방식** 유지. 2018년도는 "번호 없음"으로 통일했음.
- 선택지는 `\n① 선택지\n② 선택지\n③ 선택지\n④ 선택지` 로 줄바꿈.
- 보기형은 `\n\n<보기>\nㄱ. ...\nㄴ. ...\n\n① ㄱ, ㄴ\n② ...` 구조.
- 표가 포함된 문제는 pipe `|` 로 표현: `| 구분 | 값1 | 값2 |`.

### 1-6. 작성 편의 스크립트
Python + `python-docx` 로 정답표만 먼저 파싱해 스켈레톤 JSON 생성 → Claude 비전으로 이미지 읽으며 `questionText`/`explanation`/`tags` 채워 넣기. 참고: 2018년도 작업 시 사용한 빌드 스크립트 패턴은 Git 히스토리에서 `_build_2018_*.py` 삭제 커밋 확인.

### 1-7. 검증 스크립트 (필수 실행)

```python
import json, os
subjects = ['운동생리학','건강체력평가','운동처방론','운동부하검사',
            '운동상해','기능해부학','병태생리학','스포츠심리학']
YEAR = '2018'  # ← 새 연도로 변경
errors = []
for subject in subjects:
    path = f'data/{YEAR}_{subject}.json'
    with open(path,'r',encoding='utf-8') as f:
        data = json.load(f)
    if len(data) != 20: errors.append(f'{subject}: count != 20')
    for i, q in enumerate(data, 1):
        if q['id'] != i: errors.append(f'{subject} Q{i}: id 불일치')
        ca = q['correctAnswer']
        if isinstance(ca, int) and not (0 <= ca <= 3):
            errors.append(f'{subject} Q{i}: correctAnswer 범위')
        if isinstance(ca, list):
            if any(not (0 <= v <= 3) for v in ca):
                errors.append(f'{subject} Q{i}: 리스트 범위')
            if ca != sorted(ca):
                errors.append(f'{subject} Q{i}: 정렬 안됨')
        expected = f'images/{YEAR} {subject}/문제 ({i}).png'
        if q['questionImage'] != expected:
            errors.append(f'{subject} Q{i}: 이미지 경로 불일치')
        if not os.path.exists(q['questionImage']):
            errors.append(f'{subject} Q{i}: 이미지 파일 없음')
        for field in ('questionText','explanation','tags'):
            if not q.get(field):
                errors.append(f'{subject} Q{i}: {field} 비어있음')
        if not q['explanation'].startswith('이 문제의 정답은'):
            errors.append(f'{subject} Q{i}: explanation 접두사 오류')
print(f'{len(errors)}개 오류')
for e in errors: print(e)
```

통과 기준: **오류 0건**.

## Phase 2 — 코드/설정 수정 (6개 JS 파일)

모든 수정은 단순히 연도 배열에 새 연도를 **추가**하는 것. 배열 순서는 기존 방식 유지 (보통 최신이 앞).

| 파일 | 수정 위치 | 내용 |
|---|---|---|
| `js/utils/certificate-utils.js` | `health-manager.years` 배열 | `years: [2018, 2019, ..., 2025]` → 새 연도 추가 |
| `js/tag-search.js` | `CERT_CONFIG['health-manager'].years` | `['2018', ...]` 에 추가 |
| `js/admin/tag-manager.js` | 상단 `const years` | 동일 |
| `js/analytics/render-progress-tab-function.js` | `HEALTH_YEARS` 상수 (파일 상단) + 내부 `const years` (약 L335) — **2곳 모두** | 동일 |
| `js/quiz/quiz-core.js` | `findLatestYear()` 내 `for (let year = currentYear; year >= 2018; year--)` | 하한 연도를 새 가장 오래된 연도로 변경 |

⚠️ **`js/quiz/quiz-core.js`의 하한 연도**는 "fallback 검색 범위"를 정의. 새 연도가 과거로 추가되는 경우(예: 2018→2017) 반드시 하한을 낮춰야 fallback이 작동함.

## Phase 3 — HTML 페이지 수정 (cft.html 4곳)

`cft.html` 한 파일에서 4개 지점을 수정합니다.

1. **연도 카드 추가** (약 L1703 `years/year_2019.html` 카드 뒤)
   ```html
   <a href="years/year_2018.html" class="card">
     <div class="card__info">
       <span class="card__name">2018년</span>
       <span class="card__meta">합격률: -</span>
     </div>
     <span class="card__arrow">→</span>
   </a>
   ```
   합격률 정보가 있으면 기재.

2. **모의고사 탭 카드 추가** (약 L1733)
   ```html
   <div class="mock-year"><span class="mock-year__label">2018</span><div class="mock-year__btns"><a href="exam-new/2018_모의고사_1교시.html" class="mock-btn">1교시</a><a href="exam-new/2018_모의고사_2교시.html" class="mock-btn">2교시</a></div></div>
   ```

3. **학습분석 탭 연도 드롭다운** (약 L1800)
   ```html
   <option value="2018">2018</option>
   ```

4. **YEARS 배열 상수** (약 L2011)
   ```js
   const YEARS = [2025, 2024, 2023, 2022, 2021, 2020, 2019, 2018];
   ```

### Phase 3-B — 연도 허브 페이지 생성

`years/year_{YYYY}.html` 생성. 가장 쉬운 방법은 기존 파일 복제 후 연도 치환.

```bash
python -c "
with open('years/year_2019.html','r',encoding='utf-8') as f: s=f.read()
with open('years/year_2018.html','w',encoding='utf-8') as f: f.write(s.replace('2019','2018'))
"
```

치환으로 덮이는 것: 제목, canonical URL, og:meta, 본문 제목, 8개 과목 카드의 href + `year=` 쿼리스트링.

### Phase 3-C — 과목별 시험 페이지 생성 (10개)

`exam-new/{YYYY}_{과목}.html` 8개 + `exam-new/{YYYY}_모의고사_1교시.html`, `exam-new/{YYYY}_모의고사_2교시.html` 2개 = **10개**.

마찬가지로 기존 파일(2019 기준) 복제 후 문자열 치환. 이 파일들은 대부분 정적 UI 쉘이고 실제 데이터는 `quiz-core.js`/`mock-exam.js`가 파일명에서 연도를 파싱해 `data/{year}_{subject}.json`을 fetch하므로 **별도 로직 수정 불필요**.

```bash
python -c "
import os
subjects = ['운동생리학','건강체력평가','운동처방론','운동부하검사','운동상해','기능해부학','병태생리학','스포츠심리학']
mock = ['모의고사_1교시','모의고사_2교시']
for name in subjects + mock:
    with open(f'exam-new/2019_{name}.html','r',encoding='utf-8') as f: s=f.read()
    with open(f'exam-new/2018_{name}.html','w',encoding='utf-8') as f: f.write(s.replace('2019','2018'))
"
```

### Phase 3-D — 과목 상세 페이지에 연도 카드 추가 (8개)

`subjects/subject_{과목}.html` 8개 파일 각각 약 L415 부근의 연도 카드 목록에 **2019 카드 뒤**에 새 연도 카드 1개 추가.

⚠️ 주의: 이 8개 파일은 `&rarr;`, `&middot;` 같은 **HTML 엔티티**를 사용하는데, `subject_운동생리학.html` 1개만 유니코드 문자(`→`, `·`)를 사용함. 스크립트로 일괄 치환할 때 두 패턴 모두 처리하거나, 파일마다 올바른 형식을 확인해야 함.

- **전체정답 문제**: 2018 기능해부학 Q16 은 "전체(1·2·3·4 모두 정답)"로 출제 오류 처리됨. `correctAnswer: [0,1,2,3]` 으로 저장하고, 해설 첫 줄은 `이 문제의 정답은 1번, 2번, 3번, 4번입니다. <br/> sol) <br/> ⚠️ 전원정답 처리된 문제` 로 통일 (2021 운동상해 Q15 선례 준수).
